An **artifact** is a single, fully self-contained UI design produced by Ai Studio in response to your prompt. It is a complete HTML document — with all CSS and layout logic embedded inline — that renders immediately in your browser without any build step, external stylesheet, or JavaScript framework. Every time you submit a prompt, Ai Studio generates exactly **three artifacts in parallel**, each representing a distinct visual interpretation of your request.

---

## How artifacts are generated

Artifact generation happens in two rapid stages orchestrated by Google Gemini:

<Steps>
  <Step title="Style naming">
    Gemini receives your prompt and first generates **three conceptual style names** — short, evocative labels like *"Midnight Minimalist"*, *"Neon Brutalist"*, or *"Frosted Calm"*. These names define the creative direction for each variant before a single line of HTML is written.
  </Step>
  <Step title="Parallel HTML generation">
    Using those three style names as additional context, Gemini generates the full HTML artifact for each variant **simultaneously**. All three streams arrive at the same time, which is why you see all three cards begin populating at once rather than one after another.
  </Step>
  <Step title="Rendering">
    Each completed artifact is injected into a sandboxed `<iframe>` and rendered live in the browser. You can interact with the UI — hover states, focus rings, scroll behavior — directly in the card.
  </Step>
</Steps>

The parallel approach means you always see the full creative space of your prompt rather than just one answer. The three variants deliberately diverge: different layout structures, type hierarchies, color palettes, and materiality — even from the same prompt.

## Artifact states

Each artifact card displays a visual state badge while generation is in progress or if something goes wrong.

<AccordionGroup>
  <Accordion title="Streaming" icon="loader">
    The artifact is actively being generated. You'll see the card animate as HTML arrives from Gemini's streaming API. The iframe progressively renders the design as tokens stream in — expect the layout to "build up" visually over a few seconds.
  </Accordion>
  <Accordion title="Complete" icon="circle-check">
    Generation finished successfully. The artifact is fully interactive inside its iframe. You can hover, click, focus, and scroll within the rendered UI to evaluate it.
  </Accordion>
  <Accordion title="Error" icon="circle-x">
    Something went wrong during generation — a network interruption, a content filter trigger, or a malformed response. The card displays an error indicator and a **Retry** button so you can regenerate that single artifact without resubmitting the whole prompt.
  </Accordion>
</AccordionGroup>

## Focusing an artifact

Clicking any artifact card **focuses** it — expanding it to a larger, full-attention view where you can properly evaluate the design details, read the copy, inspect the layout, and decide what to do with it.

While an artifact is focused, the surrounding UI steps back so nothing competes for your attention. Press **Escape** or click outside the focused card to return to the three-up comparison view.

<Tip>
  Use the focused view to check details that are hard to see at card size: input field spacing, hover state styling, icon alignment, and responsive breakpoints.
</Tip>

## Artifact actions

When an artifact is focused, a set of action controls becomes available in the action bar.

<CardGroup cols={2}>
  <Card title="Favorite" icon="heart" href="/features/library">
    Press the heart icon to mark the artifact as a favorite. Favorited artifacts are flagged visually and can be filtered in the Library panel, making it easier to find your best designs across sessions.
  </Card>
  <Card title="View Source" icon="code" href="/features/export">
    Click the **SOURCE** button to open the code panel. This displays the raw HTML of the artifact — the complete, copy-pasteable document. See [Exporting Your Designs](/features/export) for full export options.
  </Card>
  <Card title="Set as Base DNA" icon="dna" href="/concepts/base-dna">
    Pin this artifact as your **Base DNA** — the reference design that all future revision prompts will build on. This is the entry point to iterative refinement. See [Base DNA](/concepts/base-dna) for the full workflow.
  </Card>
  <Card title="Retry" icon="rotate-right" href="/features/generation">
    Regenerate this specific artifact using the same prompt and the same style name. Use this when the initial generation produced an error, or when you want a fresh attempt at a variant you found unsatisfying without losing the other two.
  </Card>
</CardGroup>

## Self-contained output

Every artifact Ai Studio generates is a **complete, standalone HTML document**. This design decision has practical consequences:

- **No dependencies** — the file contains all CSS inline. There are no `<link>` tags to external stylesheets, no CDN scripts, no framework runtime to install.
- **Portable** — you can copy the HTML and paste it directly into any file, code sandbox, or CMS that accepts raw HTML.
- **Inspectable** — opening the source reveals clean, readable HTML and CSS that serves as a useful reference even if you plan to re-implement the design in a different stack.
- **Renderable anywhere** — drop the file in a browser, embed it in an iframe, or open it in a local `file://` context — it works in all three.

<Note>
  Artifacts are saved as part of their parent session in your browser's local storage. Sessions are retained automatically (up to 50 sessions) so you can revisit and re-export any artifact from your recent history without having to regenerate it. See [Sessions and History](/concepts/sessions) for details on how persistence works.
</Note>
