Every time you submit a prompt in Ai Studio, the app creates a **session** — a named container that holds your prompt, the three artifacts generated from it, and any metadata about how that generation was produced. Sessions are saved automatically and permanently accessible through the history panel, giving you a full audit trail of your design exploration without any manual save step.

---

## What is a session?

A session is the fundamental unit of work in Ai Studio. It packages everything that belongs to a single generation event:

| Field | Description |
|---|---|
| **id** | A unique identifier for the session |
| **kind** | `initial` for first-generation prompts; `revision` for prompts submitted in Base DNA revision mode |
| **prompt** | The exact text you submitted |
| **artifacts** | The three generated UI variants (each with its own id, style name, HTML, status, and favorite flag) |
| **timestamp** | When the session was created |
| **parentSessionId** | For revision sessions, the id of the session this was derived from |
| **revisionNote** | An optional note describing what changed (revision sessions) |
| **selectedArtifactIndex** | Which artifact was last focused (0, 1, or 2) |
| **brief** | The QuickBrief configuration used, if the Quick Brief tool was active when you submitted |

**Revision sessions** are linked sessions. When you have a [Base DNA](/concepts/base-dna) set and submit a revision prompt, the new session records the `parentSessionId` of the Base — creating a chain of linked sessions that represents your iteration track. You can navigate this chain to see exactly how a design evolved from first generation to final form.

## Session history

Ai Studio automatically saves every session to your browser's local storage. You don't need to click Save — as soon as a session's artifacts finish generating, it's recorded.

<Steps>
  <Step title="Open the NAV ACCESS sidebar">
    Click the **NAV ACCESS** button (or the sidebar icon) to open the navigation panel on the left side of the screen.
  </Step>
  <Step title="Navigate to HISTORY">
    Select the **HISTORY** panel within the sidebar. This lists all your saved sessions in reverse-chronological order, showing the prompt text and a thumbnail of the selected artifact for each session.
  </Step>
  <Step title="Click any session to jump to it">
    Clicking a session in the history list loads it into the main canvas. All three artifacts are restored from your browser's local storage exactly as they were when the session was saved.
  </Step>
</Steps>

## Navigating sessions

Beyond the history panel, you can move between sessions using the **left and right arrow buttons** in the main interface. These buttons step through your session history sequentially — right for newer sessions, left for older ones — without opening the history panel.

This is useful during active exploration: submit a few prompt variations, then arrow back through them to compare the results side by side in your memory before committing to a direction.

You can also navigate the revision chain of a specific design:

- In a revision session, a **parent link** in the session detail takes you back to the original base session.
- In a base session, a **revisions indicator** shows how many revision sessions were derived from it.

## Data persistence

Ai Studio stores all session data locally in your browser. Because everything is stored locally, your design work is **entirely private by default** — nothing is sent to a server or associated with an account. Your history is available immediately on every visit without any sign-in.

The trade-off is that your data lives in the browser where it was created. If you switch browsers, use a private/incognito window, or clear your browser data, your session history will not be available.

<Tip>
  Power users who want to inspect or back up their data can find session records in their browser's local storage. Ai Studio stores session history under the key `alchemy_history` — you can view or export this data using your browser's developer tools (Application → Local Storage).
</Tip>

## Session limits

Ai Studio retains a maximum of **50 sessions** in history. When you exceed this limit, the oldest session is automatically removed to make room for the new one. This keeps local storage usage predictable and prevents the app from slowing down over time.

<Accordion title="What gets deleted when a session is removed?">
  When the oldest session is pruned, its session record and all associated artifact HTML are removed from your browser's local storage. If you had a Base DNA reference pointing to an artifact in the deleted session, that reference will no longer resolve. To avoid this, save important artifacts to the Library before they age out of the 50-session window.
</Accordion>

To preserve a design permanently — beyond the 50-session rolling window — add it to the **Library**. Library items are stored separately from the session history and are never automatically deleted. See [Organizing Your Work](/guides/organizing-work) for how to use the Library and folder system.

<Warning>
  Clearing your browser's site data (cookies, cache, and local storage) through your browser settings will permanently erase your entire session history, all artifact HTML, and your Library folder structure. There is no cloud backup and no way to recover this data once deleted. Before clearing browser data, export any designs you want to keep using the **SOURCE** panel or the export tools described in [Exporting Your Designs](/features/export).
</Warning>
